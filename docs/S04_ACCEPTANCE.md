# S04 — локальная приёмка

Дата: 24 июля 2026.

## Результат

Контрольный рубеж пройден: одноразовое приглашение атомарно создаёт сотрудника, назначения на объекты и серверную сессию.

## Подтверждено

- только подрядчик создаёт и отзывает приглашения;
- приглашение ограничено ролью рабочего или бригадира, организацией, выбранными объектами и сроком 1–14 дней;
- открытый токен показывается только в созданной ссылке, PostgreSQL хранит SHA-256-хэш;
- публичный просмотр не раскрывает автора и внутренний комментарий;
- onboarding сохраняет ФИО и специализацию, нормализует уникальный телефон и хэширует пароль через scrypt;
- регистрация одной транзакцией создаёт пользователя, членство и объектные назначения и помечает ссылку использованной;
- новая HttpOnly-сессия сразу открывает кабинет сотрудника только с разрешёнными объектами;
- использованная, отозванная и истёкшая ссылка отклоняются;
- новый сотрудник появляется в команде подрядчика с фактическими ФИО, телефоном, специализацией и назначением.

## Проверки

- `npm run check` — успешно;
- `npm test` — 11/11 тестов;
- `npm run db:migrate:verify` — 5 миграций применяются на чистой схеме и безопасно пропускаются при повторном запуске;
- `npm run build` — production-сборка успешна;
- сквозной браузерный проход — создание ссылки, onboarding, автоматический вход, кабинет рабочего и появление в команде;
- адаптивная проверка на ширине 360 px — горизонтального переполнения нет;
- browser console — ошибок и предупреждений нет;
- созданные для приёмки пользователь и приглашения удалены.

Публичное размещение приложения, SMS и проверка QR на физических устройствах не входили в этот срез.
