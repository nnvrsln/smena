# Production-срез S03 — эксплуатационный фундамент

Дата старта: 23 июля 2026
Статус: **в работе**

## Цель

Текущий production web, API и PostgreSQL должны собираться, проверяться, мигрироваться, диагностироваться и восстанавливаться воспроизводимо вне локальной машины разработчика.

## Контрольный рубеж

`Чистый staging-контур поднимается из конфигурации репозитория, применяет миграции, проходит проверки и восстанавливает тестовую резервную копию без потери изоляции организаций.`

## Входит

- явные шаблоны переменных для development, test, staging и production без секретов;
- одна воспроизводимая команда применения миграций;
- CI для проверки типов, API-тестов и production-сборки;
- health-check без раскрытия секретов и персональных данных;
- единый безопасный ответ API при непредвиденной ошибке;
- структурированные серверные логи с request ID;
- команды создания и тестового восстановления резервной копии PostgreSQL;
- runbook staging-запуска, диагностики и восстановления;
- существующий тест tenant isolation как обязательная часть рубежа.

## Не входит

- выбор и оплата облачного провайдера;
- публикация production и настройка домена;
- хранение реальных секретов в Git;
- новая продуктовая функциональность;
- offline/outbox, push и уведомления;
- задачи, проблемы, файлы и месячное закрытие;
- изменение визуального прототипа.

## Критерии приёмки

- `npm run check` завершается успешно;
- `npm test` поднимает изолированную тестовую схему и проходит полностью;
- `npm run build` создаёт production-сборку;
- миграции применяются одной документированной командой к пустой БД;
- CI повторяет check, test и build на чистом runner;
- health-check различает готовность API и доступность PostgreSQL;
- непредвиденная ошибка возвращает безопасный ответ с request ID;
- серверные логи позволяют связать запрос, ошибку и HTTP-ответ без вывода cookie, пароля или токена;
- резервная копия создаётся и восстанавливается в отдельную тестовую БД по runbook;
- после восстановления проходит проверка tenant isolation;
- в репозитории отсутствуют staging/production-секреты.

## Условие завершения

S03 завершён только после воспроизводимого контрольного прохода на чистом staging-контуре. Реальные проверки QR-камеры Android/iPhone и fallback-декодера выполняются на этом контуре перед релизной приёмкой, но не расширяют инженерный объём S03.
